## 一、项目概述

### 1.1 项目背景

蒙古族服饰是国家级非物质文化遗产，承载着丰富的民族文化与历史记忆。本平台旨在通过数字藏品的形式，对蒙古族传统服饰文化进行数字化保护与传播。用户可在平台上浏览、购买、收藏蒙古服饰相关的数字藏品（包括服饰图片、纹样、工艺视频、3D模型等）。平台模拟区块链上链流程，为每件藏品生成唯一链上凭证（哈希值），但实际数据存储在中心化数据库中。

### 1.2 核心目标

- 提供蒙古服饰非遗数字藏品的发行、展示、购买的完整流程
- 模拟区块链存证，给用户"上链"体验
- 支持用户间转赠功能
- 安全可靠的支付与账户体系

## 四、功能模块

### 3.1 用户模块

#### 3.1.1 注册与登录
- 邮箱 + 密码注册/登录
- 登录后自动生成"数字钱包地址"（模拟，格式：`0x` + 40位十六进制字符串）

#### 3.1.2 用户信息
- 头像、昵称、个人简介
- 实名认证（姓名 + 身份证号，购买/转赠前必须完成）
- 数字钱包地址展示
- 钱包余额（支持模拟充值）

#### 3.1.3 我的藏品
- 展示用户持有的所有藏品（卡片网格视图）
- 支持按时间/名称/系列排序
- 藏品详情：图片、名称、编号、链上哈希、获取时间、获取方式（购买/转赠/空投）

---

### 3.2 藏品模块

#### 3.2.1 藏品信息结构

| 字段 | 说明 |
|------|------|
| 藏品名称 | 最长30字符 |
| 藏品封面 | 主图，支持 JPG/PNG/GIF/WebP |
| 藏品文件 | 原始高清文件（图片/音频/视频/3D） |
| 系列名称 | 所属藏品系列 |
| 创作者 | 创作者名称及头像 |
| 发行数量 | 限量总份数 |
| 编号 | 如 #0001/#1000 |
| 价格 | 单价（人民币），精确到分 |
| 发售时间 | 定时开售 |
| 藏品描述 | 富文本，最长2000字 |
| 链上哈希 | 模拟生成的唯一哈希值 |
| 链上时间 | 模拟上链时间 |
| 合约地址 | 模拟智能合约地址 |
| 区块高度 | 模拟区块编号 |

#### 3.2.2 藏品状态流转

```
创建(草稿) → 审核中 → 已通过 → 待发售 → 发售中 → 已售罄/已下架
                     → 已拒绝
```

#### 3.2.3 模拟上链机制

藏品通过审核后，系统自动生成以下模拟链上信息：
- **链上哈希**：对藏品元数据（名称 + 创作者 + 发行时间 + 随机盐）进行 SHA-256 哈希，生成 `0x` + 64 位十六进制字符串，作为藏品的唯一链上凭证
  ```javascript
  const crypto = require('crypto')
  const salt = crypto.randomBytes(16).toString('hex')
  const raw = `${name}:${creator}:${publishTime}:${salt}`
  const chainHash = '0x' + crypto.createHash('sha256').update(raw).digest('hex')
  ```
- **合约地址**：`0x` + 随机生成 40 位十六进制，模拟智能合约部署地址
- **区块高度**：全局自增计数器（起始值 18000000），每次铸造 +1，模拟区块链出块
- **链上时间**：审核通过的时间戳，精确到秒
- **TokenID**：系列内自增编号，标识该藏品在系列中的唯一序号

### 3.4 交易模块

#### 3.4.1 购买流程

```
选择藏品 → 确认订单 → 钱包余额支付 → 支付成功 → 藏品入库（生成编号）→ 模拟上链确认
```

- 用户通过平台钱包余额完成支付（模拟支付，不接入真实支付渠道）
- 用户可在个人中心进行模拟充值（直接增加余额）
- 余额不足时提示充值

- 发售时间精确到秒，未到时间显示倒计时
- 限购规则：每人每款限购 N 件（可配置，默认1件）
- 高并发抢购场景需排队机制
- 订单15分钟内未支付自动取消
- 支付成功后显示"上链中"动画（2-3秒），然后显示"上链成功"

#### 3.4.2 订单管理
- 订单状态：待支付 → 已支付 → 已完成 / 已取消 / 已退款
- 订单详情：藏品信息、支付金额、交易时间、链上凭证
- 订单15分钟内未支付自动取消

#### 3.4.3 转赠功能
- 实名认证用户之间可互相转赠
- 转赠需输入对方钱包地址
- 转赠后生成新的流转记录
- 转赠冷却期：购买后 72 小时内不可转赠（可配置）
- 转赠过程显示"链上转移中"动画

## 五、非功能需求

### 4.1 性能要求
- 首页加载时间 ≤ 2秒
- 抢购接口 QPS ≥ 1000
- 支付回调处理 ≤ 500ms
- 藏品列表接口响应 ≤ 200ms

### 4.2 安全要求
- 接口签名验证，防止篡改
- 抢购防刷：频率限制 + 验证码 + 设备指纹
- 敏感数据加密存储（身份证号、手机号）
- SQL注入/XSS防护
- HTTPS全站加密

### 4.3 可用性
- 系统可用性 ≥ 99.9%
- 数据每日自动备份
- 关键操作日志记录

### 4.4 兼容性
- 浏览器：Chrome、Safari、Firefox、Edge 最近两个大版本
- 移动端：iOS 14+、Android 10+
- 屏幕适配：375px ~ 1920px 响应式

## 六、模拟上链技术方案摘要

由于不接入真实区块链，核心在于**让用户感知到"上链"体验**：

| 环节 | 模拟方式 |
|------|----------|
| 链上哈希 | SHA-256(藏品元数据 + 随机盐) |
| 合约地址 | 固定前缀 + 随机十六进制（每个系列一个） |
| 区块高度 | 全局递增计数器 |
| 上链确认 | 2-3秒延迟动画 + 状态变更 |
| 流转记录 | 数据库记录所有权变更历史 |
| 链上查询 | 平台内提供"区块链浏览器"页面（展示本地数据） |
| TokenID | 系列内自增 |
